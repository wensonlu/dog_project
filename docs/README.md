## 📋 文档命名规范

### 文件命名
```
[日期]-[类型]-[标题].md

示例：
2026-02-12-audit-交互设计评估报告.md
2026-02-12-prd-领养功能需求文档.md
2026-02-12-feature-搜索筛选功能规格.md
```

### 类型标识
- `prd` - 产品需求文档
- `req` - 需求文档
- `audit` - 评估/审计报告
- `flow` - 流程图文档
- `spec` - 规格说明
- `guide` - 指南/规范
- `plan` - 计划/路线图

## 📝 各目录说明

### 01-product/ - 产品文档
存放产品层面的文档：
- 产品愿景和目标
- 产品路线图 (Roadmap)
- 竞品分析
- 用户画像
- 版本更新日志

### 02-requirements/ - 需求文档
存放需求分析文档：
- 用户故事 (User Stories)
- 功能需求列表
- 非功能需求 (性能、安全等)
- API 接口契约

### 03-interaction/ - 交互设计
存放交互设计相关文档：
- 交互评估报告
- 用户流程图
- 信息架构图
- 原型/线框图
- 交互规范

### 04-design/ - 视觉设计
存放视觉设计相关文档：
- UI 设计规范
- 组件库文档
- 色彩/字体规范
- 设计资源文件

### 05-improvements/ - 改进记录
存放项目改进记录：
- **planned/** - 已规划待实施的改进
- **in-progress/** - 正在进行的改进
- **completed/** - 已完成的改进

每个改进项应包含：
- 问题描述
- 改进方案
- 实施进度
- 验收标准

### 06-archive/ - 归档文档
存放过期或废弃的文档：
- 旧版本需求
- 废弃的设计方案
- 历史决策记录

## ✅ 文档编写规范

### Markdown 格式
- 使用标准 Markdown 语法
- 标题层级不超过 4 级
- 代码块标注语言类型
- 表格用于展示结构化数据

### 文档模板

#### 评估报告模板
```markdown
# [标题]

## 基本信息
- 评估日期：
- 评估人：
- 版本：

## 评估范围
...

## 发现的问题
...

## 整改建议
...

## 优先级汇总
...
```

#### 需求文档模板
```markdown
# [功能名称] 需求文档

## 背景
...

## 目标
...

## 用户故事
...

## 功能规格
...

## 验收标准
...
```
