# 宠物领养平台创新点调研与PRD（v1）

**日期**: 2026-04-17  
**负责人**: Leader（基于 senior-pm-worker 既有MVP文档补充）

## 1. 创新点调研（可在当前架构落地）

### 1.1 创新点A：可信领养评分卡（Adoption Readiness Score）
- 描述：在申请页展示“资料完整度 + 条件匹配度”评分，给出可操作建议（如补充居住信息、养宠经验）。
- 用户价值：降低盲投申请，提高通过率预期和体验可控感。
- 业务价值：减少无效申请，降低审核成本。
- 成本评估：中。
- 风险：评分规则过于黑箱会引发不信任，需要可解释项。

### 1.2 创新点B：领养流程时间线（Timeline）
- 描述：用户侧提供“已提交/审核中/已通过/已拒绝”可视化时间线，并展示最近动作与建议下一步。
- 用户价值：减少焦虑和重复咨询。
- 业务价值：降低客服压力，提高留存。
- 成本评估：低。
- 风险：状态来源必须与后端一致，避免前后状态不一致。

### 1.3 创新点C：高意向匹配提醒（Preference Match Alert）
- 描述：根据收藏行为 + 筛选偏好，新增“可能适合你”的轻量提醒（站内消息优先，不做复杂推荐模型）。
- 用户价值：提升回访与申请转化。
- 业务价值：提高有效申请量。
- 成本评估：中。
- 风险：先做规则型触发，避免过早引入复杂推荐。

### 1.4 创新点D：审核反馈模板化（Admin Feedback Template）
- 描述：管理员拒绝申请时可选择标准原因模板并附建议，自动同步到用户消息中心。
- 用户价值：获得明确改进方向。
- 业务价值：降低负反馈，提升二次申请成功率。
- 成本评估：低。
- 风险：模板措辞需要合规与友好。

### 1.5 创新点E：冷启动信任增强（Profile Trust Badge）
- 描述：对完成关键资料的用户显示“资料已完善”标识。
- 用户价值：用户知道自己是否达到基本门槛。
- 业务价值：提高资料质量与审核效率。
- 成本评估：低。
- 风险：仅用于信息提示，不做绝对资格判定。

## 2. MVP功能边界（本轮）

### 2.1 本轮纳入（P0 + 部分P1）
- P0：浏览、收藏、申请提交、审核、消息通知闭环。
- 创新增量（本轮纳入）
  - 时间线（创新点B）
  - 审核反馈模板化（创新点D）
  - 资料完善标识（创新点E，简版）

### 2.2 本轮不纳入（延后）
- 高意向匹配提醒（创新点C）
- 申请评分卡完整版本（创新点A，先保留埋点与字段）

## 3. 用户故事与验收标准

### 3.1 用户故事
- 作为领养用户，我希望看到申请进展时间线，这样我知道当前卡在哪一步。
- 作为领养用户，我希望被拒绝时能看到明确原因与建议，这样我可以修改后再次申请。
- 作为管理员，我希望快速选择拒绝原因模板，提高审核效率且沟通一致。

### 3.2 验收标准（DoD）
- 时间线：用户在个人中心可查看每个申请的状态节点与更新时间。
- 反馈模板：管理员拒绝申请时必须选择至少一个原因模板并可追加备注。
- 通知同步：审核动作后，消息中心在可接受延迟内展示一致内容。
- 可回归：新增功能具备最小可执行测试用例（接口 + 前端关键流）。

## 4. 非功能要求
- 稳定性：不得破坏既有浏览/收藏/申请主流程。
- 可观测性：关键动作写入日志并可追踪申请状态流转。
- 兼容性：前后端接口字段新增保持向后兼容。

## 5. 两周里程碑
- Week1：PRD冻结、前后端方案评审、数据结构与接口草案冻结。
- Week2：开发完成、联调测试、缺陷收敛、PR合并。

## 6. 输出要求给各角色
- 后端：基于本PRD提交数据模型与接口方案文档。
- 前端：基于本PRD提交页面与状态管理方案文档。
- QA：基于本PRD补充时间线/反馈模板/通知一致性测试用例。
