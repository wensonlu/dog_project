# AI话题生成功能设计文档

**设计日期**: 2026-04-09
**设计目标**: 简化论坛发帖流程，通过AI辅助生成降低用户发帖门槛

---

## 一、产品定位

### 核心理念
为论坛增加AI辅助发帖功能，让用户通过输入关键词即可快速生成完整话题内容，降低发帖门槛，提升内容生产效率。

### 目标用户
- **新手用户**：不知道如何组织语言，需要AI帮助
- **忙碌用户**：想分享但时间有限，需要快速生成
- **传统用户**：习惯自己写作，可继续使用原表单

### 产品价值
1. **降低门槛**：从5个字段简化为关键词输入
2. **提升效率**：AI生成耗时仅5-10秒
3. **保持控制**：用户可编辑所有生成内容
4. **灵活选择**：提供AI模式和传统模式

---

## 二、用户流程设计

### 整体流程

用户进入CreateTopic页面后，看到两个选项：
- **"AI帮我写"** → 极简模式（推荐）
- **"我自己写"** → 原有表单模式

### AI模式流程

```
1. 点击"AI帮我写"按钮
         ↓
2. 输入关键词（如：金毛领养经验、第一次养狗）
         ↓
3. 点击"生成话题内容"
         ↓
4. 显示加载动画（5-10秒）
         ↓
5. 卡片预览展示：
   - 标题（可编辑）
   - 内容（可编辑）
   - 分类（可重新选择）
   - 标签（可编辑）
         ↓
6. 可选：添加图片（最多9张）
         ↓
7. 点击"发布话题"
```

### 传统模式流程
保持现有的完整表单流程不变，给需要精确控制的用户使用。

## 三、技术架构设计

### 前端组件结构

```jsx
<CreateTopic>
  {/* 模式选择区 */}
  <ModeSelector>
    - 按钮A: "AI帮我写" (渐变色背景，主要推荐)
    - 按钮B: "我自己写" (普通按钮，次要选项)
  </ModeSelector>

  {/* AI模式 - 关键词输入区 */}
  {mode === 'ai' && !generatedContent && (
    <KeywordsInput>
      - 输入框: placeholder="输入关键词，如：金毛领养经验、第一次养狗注意事项"
      - 提示文本: "多个关键词用逗号或空格分隔"
      - 生成按钮: "生成话题内容"
    </KeywordsInput>
  )}

  {/* 加载动画 */}
  {isGenerating && (
    <LoadingAnimation>
      - 动画: "AI正在为你创作内容..."
      - 进度提示: "预计5-10秒"
    </LoadingAnimation>
  )}

  {/* 卡片预览区 */}
  {generatedContent && (
    <TopicPreviewCard>
      - 标题编辑: textarea可编辑
      - 内容编辑: textarea可编辑
      - 分类选择: 按钮组（领养经验、日常分享、求助问答）
      - 标签编辑: 输入框，逗号分隔
      - 图片上传区: 九宫格上传组件（复用现有逻辑）
      - 操作按钮: "重新生成" | "发布话题"
    </TopicPreviewCard>
  )}

  {/* 传统模式 - 保持原有表单 */}
  {mode === 'manual' && <OriginalForm />}
</CreateTopic>
```

## 四、AI生成逻辑

### 工具函数：generateTopicContent

在 `backend/utils/ai.js` 中新增函数。

**Prompt设计：**
```javascript
根据用户输入的关键词，生成一个完整的论坛话题。

关键词：${keywords}

请生成以下内容（以JSON格式返回）：
{
  "title": "吸引人的话题标题（10-30字）",
  "content": "话题正文内容（200-500字，鼓励互动）",
  "category": "领养经验|日常分享|求助问答",
  "tags": ["标签1", "标签2", "标签3"]
}

生成要求：
- 标题要吸引人，能引发讨论
- 内容要真诚、有温度，鼓励其他用户参与互动
- 分类要根据关键词内容智能判断
- 标签要准确概括话题主题（3-5个）
```

**分类判断规则：**
- **领养经验**：分享领养故事、经验、建议
- **日常分享**：宠物日常、趣事、照片分享
- **求助问答**：提问、寻求帮助、咨询问题

**成本控制：**
- 单次生成约$0.03-0.05（使用GLM-5）
- 提供重新生成功能，最多3次
- 显示生成成本提示："本次AI生成消耗约0.03美元"

## 五、错误处理和边界情况

### 错误处理策略

**1. AI生成失败处理：**
- **网络超时：** 显示"AI服务响应超时，请稍后重试"，提供重试按钮
- **API错误：** 显示具体错误信息，引导用户使用传统模式
- **内容解析失败：** 显示"AI生成内容格式异常，请重新生成"，记录错误日志

**2. 用户输入验证：**
```javascript
// 关键词验证
- 最小长度：3个字符
- 最大长度：200个字符
- 内容限制：只能包含中英文、数字、逗号、空格
- 提示：输入不合规时显示红色边框+错误提示
```

**3. 生成次数限制：**
```javascript
// 防止滥用和成本失控
const GENERATION_LIMITS = {
  maxRetries: 3,           // 每个话题最多重新生成3次
  cooldownMinutes: 5,      // 超限后冷却时间
  sessionMax: 10           // 每个用户每天最多生成10个话题
};

// 超限提示："您已生成3次，建议编辑现有内容或使用传统模式"
```

**4. 内容安全检查：**
```javascript
// AI生成内容后，进行敏感词过滤（可选）
- 使用Supabase的敏感词过滤功能
- 或在前端显示"内容审核中"提示
- 发现问题时引导用户修改
```

## 六、交互细节设计

### 关键交互点

1. **关键词输入提示：** 显示示例关键词气泡，帮助用户理解如何输入
2. **生成按钮禁用逻辑：** 关键词为空时禁用，生成中禁用
3. **重新生成限制：** 显示"剩余2次重新生成机会"，最多3次
4. **成本透明：** 卡片底部显示"本次生成成本：约$0.03"
5. **无缝切换：** 用户可随时切换回传统模式（已生成的内容会保留）

## 九、风险评估

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| AI生成质量不稳定 | 用户体验差 | 提供重新生成+编辑功能 |
| 成本超预算 | 财务压力 | 每日生成次数限制+成本监控 |
| 关键词输入不明确 | 生成内容偏离 | 显示输入示例引导用户 |
| 网络延迟影响体验 | 用户流失 | 优化加载动画，提供传统模式备选 |
