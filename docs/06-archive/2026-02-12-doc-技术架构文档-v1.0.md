## 3. 数据库 Schema

### 3.1 核心数据表

#### `profiles` - 用户资料
```sql
CREATE TABLE public.profiles (
  id UUID PRIMARY KEY REFERENCES auth.users ON DELETE CASCADE,
  email TEXT NOT NULL,
  full_name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**说明：** 与 Supabase Auth 用户表关联，存储扩展用户信息。

#### `dogs` - 宠物信息
```sql
CREATE TABLE dogs (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  name TEXT NOT NULL,
  age TEXT NOT NULL,
  breed TEXT NOT NULL,
  location TEXT NOT NULL,
  image TEXT NOT NULL,
  gender TEXT DEFAULT '公',
  description TEXT,
  traits JSONB DEFAULT '[]',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**说明：** 核心宠物数据表，存储待领养宠物信息。

#### `favorites` - 收藏
```sql
CREATE TABLE favorites (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  user_id UUID DEFAULT auth.uid(),
  dog_id BIGINT REFERENCES dogs(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, dog_id)
);
```

**说明：** 用户收藏夹，多对多关系表。

#### `applications` - 领养申请
```sql
CREATE TABLE applications (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  user_id UUID DEFAULT auth.uid(),
  dog_id BIGINT REFERENCES dogs(id) ON DELETE CASCADE,
  full_name TEXT NOT NULL,
  phone TEXT NOT NULL,
  address TEXT NOT NULL,
  has_pets BOOLEAN DEFAULT FALSE,
  housing_type TEXT NOT NULL,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**状态说明：**
- `pending` - 待审核
- `approved` - 已通过
- `rejected` - 已拒绝

#### `messages` - 消息
```sql
CREATE TABLE messages (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  user_id UUID DEFAULT auth.uid(),
  sender_name TEXT NOT NULL,
  content TEXT NOT NULL,
  image_url TEXT,
  is_unread BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**说明：** 系统消息表，用于通知用户申请状态等。

#### `dog_submissions` - 用户提交（待审核）
```sql
CREATE TABLE dog_submissions (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  user_id UUID NOT NULL,
  name TEXT NOT NULL,
  age TEXT NOT NULL,
  breed TEXT NOT NULL,
  location TEXT NOT NULL,
  image TEXT NOT NULL,
  gender TEXT DEFAULT '公',
  description TEXT,
  traits JSONB DEFAULT '[]',
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  reviewed_at TIMESTAMPTZ,
  reviewer_id UUID
);
```

**说明：** 用户提交的送养申请，需管理员审核。

### 3.2 论坛数据表

#### `forum_topics` - 话题
```sql
CREATE TABLE forum_topics (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  user_id UUID NOT NULL REFERENCES auth.users(id),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  category TEXT NOT NULL DEFAULT '日常分享',
  tags JSONB DEFAULT '[]',
  images JSONB DEFAULT '[]',
  likes_count INTEGER DEFAULT 0,
  comments_count INTEGER DEFAULT 0,
  views_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### `forum_comments` - 评论
```sql
CREATE TABLE forum_comments (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  topic_id BIGINT NOT NULL REFERENCES forum_topics(id),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  content TEXT NOT NULL,
  likes_count INTEGER DEFAULT 0,
  replies_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### `forum_replies` - 回复（评论的评论）
```sql
CREATE TABLE forum_replies (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  comment_id BIGINT NOT NULL REFERENCES forum_comments(id),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  content TEXT NOT NULL,
  likes_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 点赞表
- `forum_topic_likes` - 话题点赞
- `forum_comment_likes` - 评论点赞
- `forum_reply_likes` - 回复点赞

**说明：** 三个独立的点赞表，使用 `UNIQUE(topic_id, user_id)` 防止重复点赞。

## 5. RLS 安全策略

### 5.2 服务角色密钥使用

后端使用 `SUPABASE_SERVICE_ROLE_KEY` 绕过 RLS 执行以下操作：

1. **管理员操作**
   - 查看所有领养申请
   - 审批/拒绝宠物提交
   - 发送系统消息

2. **跨用户操作**
   - 当申请被批准时，自动发送消息给申请者
   - 统计全局数据

## 11. 故障排查

### 常见问题

**Q: Supabase 连接失败？**
```
检查环境变量：
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
```

**Q: CORS 错误？**
```
检查 backend/vercel.json 中的 headers 配置，
确保 Access-Control-Allow-Origin 匹配前端域名。
```

**Q: 图片上传失败？**
```
1. 检查 Storage bucket 'dog-images' 是否存在
2. 检查 Storage 的 RLS 策略
3. 确认文件大小限制
```
